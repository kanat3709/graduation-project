# graduation-project

1. サービス概要 
本サービスは、旅行の思い出を「場所」と「感情」で記録・振り返ることができる旅行ログアプリである。忙しい社会人が、写真を整理できず記憶が曖昧になってしまう課題に対し、簡単な操作で体験を残せる仕組みを提供する。 

2. このアイデアはどこから生まれたか 
2-1. きっかけとなった体験・感情 
本サービスのアイデアは、自身の留学および旅行体験から生まれた。 
2014年〜2015年にかけてカナダへ留学し、帰国後しばらくしてから現地で撮影した写真を見返していた際、当時の記憶や感情が鮮明に蘇る体験をした。 
また、2025年に金沢旅行から帰宅した後も、同様に写真を見返しながら「楽しかった」「また行きたい」といった感情や、新しい土地に足を踏み入れたときのワクワク感や高揚感を思い出した。 
一方で、時間が経つにつれて「この写真はどこで撮ったのか」「そのとき何を感じていたのか」が曖昧になっていくことにも気づいた。 
本来であれば、思い出はより鮮明に残しておきたいものであるにもかかわらず、写真が整理されていないことで、記憶や感情が徐々に薄れていくことに違和感や哀愁を覚えた。 

2-2. なぜそれが「気になった」のか 
この課題が特に気になった背景には、自身の性格や過去の経験が影響していると考える。 
もともと好奇心が旺盛で、新しい場所や体験に価値を感じやすい性格である一方、几帳面な一面もあり、物事を整理して残しておきたいという意識がある。 
過去には、大切にしていた写真データが埋もれてしまい、後から見つけられなくなった経験があり、その際に強い喪失感や後悔を感じた。 
また、自分にとって旅行は単なる移動や観光ではなく、「その場所で何を感じたか」という体験そのものに価値があると考えている。 
そのため、時間の経過とともに感情や記憶が薄れてしまうことに対して、「もったいない」「きちんと残したい」という思いが強くなった。 

3. 課題の整理 
3-1. 表に見えている困りごと 
旅行の写真はスマートフォンに保存されているが、後から見返したときに「どこで撮った写真なのか」が分かりづらい 

＜なぜ？＞ 
・写真は撮影しているものの、整理できていない 
・時間が経つにつれて、訪れた場所や時間の記憶が曖昧になる 
・その場所で感じた感情や体験を思い出せなくなる 
・自分だけでなく、他の人の旅行体験も見てみたいというニーズがある 

3-2. 本当に解決したい課題 
上記の課題に対して「なぜそれが困るのか」を深掘りした結果、以下の本質的な問題が見えてきた。 
・写真が整理されていないことで、必要な情報にすぐアクセスできず、結果として見返す機会が減ってしまう 
・社会人として日々の業務に追われる中で、思い出を振り返る時間や余裕がなくなっている 
・その結果、大切な体験や感情が記録されないまま、時間とともに失われてしまう 

さらに、人とのつながりという観点では、 
・社会人になると新しい出会いや交流の機会が減り、趣味でつながる関係が築きにくい 
・思い出を共有する相手が限られてしまう 
といった課題もある。 

これらを踏まえ、本当に解決したい課題は以下である。 
→忙しい日常の中でも、旅行の思い出や感情を無理なく記録・振り返りをできる状態をつくること 
→「思い出」をきっかけに、他者とのゆるやかなつながりを生み出すこと 

3-3. 解決策のイメージ 
例：金沢旅行の場合 
- 地図上に「金沢」のピンが立つ 
- 写真：兼六園の写真 
- 感情タグ：「感動した」「また行きたい」 
- 一言メモ：「夏の兼六園、想像以上に美しかった」 
→ 後から見返したときに、写真だけでなく「その時の気持ち」まで蘇る 

4-1. 想定ユーザー 
年齢設定 
・年齢：20代後半〜30代 
・理由： 
　この年代は、経済的・時間的に旅行に行く余力がある一方で、仕事による疲労や忙しさから、思い出の整理や記録に十分な時間を割けない傾向がある。 
そのため、「簡単に記録できる仕組み」に対するニーズが高いと考えた。 

生活・状況設定 
・生活・状況：社会人・一人暮らし 
・理由： 
　日常的に仕事中心の生活になりやすく、リフレッシュ手段として旅行の価値が高い層である。 
　一方で、思い出を共有する機会が限られるため、アプリを通じて振り返りや共有ができる価値がある。 

環境設定 
・環境：PC（将来的にスマートフォンアプリを想定） 
・理由： 
　開発初期はPCブラウザでの利用を想定するが、実際の利用シーンはスマートフォンでの写真撮影・投稿が中心となる。 
　そのため、将来的にはスマートフォンで「撮影→即記録」ができる体験を重視する。 

4-2. 自分とユーザーの距離 
・選択：自分自身 
・理由： 
　本アプリの課題は、自身の旅行体験から生まれたものである。 
　実際に「写真はあるが整理できていない」「時間が経つと感情や記憶が薄れてしまう」という悩みを感じており、同じような課題を持つユーザーに対して、より解像度高くニーズを捉えられると考えたため。 

4-3. 実際に使われる可能性について 
・最初に使ってくれそうな人： 
　旅行が好きで、写真をよく撮るが整理できていない社会人の友人や、自分自身 
・どんな場面で思い出すか： 
　旅行から帰ってきた直後や、写真フォルダを見返しているとき 
　また、次の旅行の計画を立てる際 
・代わりに使っているもの： 
　Instagram、Googleフォト、スマートフォンの写真フォルダ 

・置き換えてまで使う理由： 
　本アプリの初期ユーザーとしては、友人や家族、また旅行が好きで日常的に写真を撮る人を想定している。 
　これらのユーザーは、旅行中や旅行から帰宅した直後に「思い出を残したい」と感じたタイミングや、日常生活の中で疲れたときに過去の写真を見返したくなったとき、また次の旅行を計画する際に本アプリを思い出すと考えられる。 
　現在は、Instagram やGoogleフォト、スマートフォンの写真フォルダを用いて写真の保存や共有が行われている。 
　しかし、これらのサービスは「保存」や「共有」には優れている一方で、「どこで」「どのような気持ちで体験したか」を整理して振り返る体験は十分に提供されていない。 
　本アプリでは、写真に加えて位置情報（地図）と感情を紐づけて記録することで、単なる写真の保存ではなく「体験そのもの」を振り返ることができる。 
　そのため、既存サービスでは得られない価値を提供でき、置き換えて利用される可能性があると考える。 

4-4. 導入・継続の流れ 
・きっかけ： 
　旅行後、「写真を整理したい」「思い出を残したい」と感じたタイミング 

・最初の一歩： 
　Googleアカウントなどで簡単にログインし、写真をアップロード 
　→ 位置情報を自動取得 or 手動で地図に紐付け 
 
・継続理由： 
　- 旅行のたびに記録することで、自分だけの「思い出マップ」が完成していく 
　- 過去の旅行を振り返る楽しさがある 
　- 他ユーザーの旅行を見ることで、新しい行き先の発見につながる 

5-1. 似たサービス 
・サービス名： Instagram 
・URL：https://www.instagram.com/
・できること： 
　写真・動画の投稿、共有、いいね・コメント機能、場所を追加 
・特徴： 
　手軽に写真を共有できるSNSであり、拡散性や他者とのコミュニケーションに強みがある 

・サービス名： Day One(日記アプリ) 
・URL：https://dayoneapp.com/ 
・できること： 
　日記に位置情報を紐付け、写真を添付、カレンダー・リスト・マップでの振り返り 
・特徴： 
　個人の記録に特化しており、日々の出来事や感情を丁寧に蓄積できる 

5-2. それでも作りたい理由 
Instagram は手軽に写真を共有できる一方で、投稿が時系列に流れていくため、過去の旅行体験を「場所」や「感情」と紐づけて振り返ることには向いていないと感じた。 
また、フォロー・フォロワーといった人間関係を前提としたつながりが中心であり、「旅行体験そのもの」や「写真」をきっかけに、よりゆるく他者とつながることは難しいと感じた。 
一方、Day One は、位置情報や写真とともに感情を記録できる点で本サービスの方向性に近いが、基本的に個人利用を前提としており、他者との共有やつながりの要素が弱い。 
そのため、場所や感情といった体験ベースで思い出を蓄積し、同じような興味や体験を持つ人とゆるやかにつながれるサービスを作りたいと考えた。

　旅行後に写真を放置せず、簡単に整理・記録するようになる

8. 懸念点（これ超重要） 
懸念点① 
・問題： 
　投稿が面倒で継続されない可能性 
・理由： 
　ターゲットは「忙しい社会人」であり、記録の手間が大きいと使われなくなる 
・対策： 
　- 写真アップロードだけで最低限成立する設計にする 
　- 位置情報を自動取得する 
　- 入力項目を極限まで減らす 
・最低限の投稿フロー： 
　1. 写真を選ぶ 
　2. 位置情報を自動取得 or 手動で地図に紐付け 
　3. 感情タグを1つ選ぶ（任意） 
　→ これだけで投稿完了 

・継続のための工夫： 
　- 旅行後に「まとめて投稿」できる機能 
　- 過去の投稿を見返すと「○ヶ月前、○年前の思い出」が表示される
